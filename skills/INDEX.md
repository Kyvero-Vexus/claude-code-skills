# Skills Index

## Stable

- **clim-spec**
  - Build CLIM applications on McCLIM using the right mix of frames, panes, commands, presentations, gadgets, formatted output, graphics, redisplay, and the rest of the CLIM 2 feature surface, with the spec as semantic ground truth.
  - Use when: the user wants to build, extend, debug, or understand the practical feature set of CLIM/McCLIM, including broad "what all does CLIM cover?" questions.
  - Do not use when: the question is generic Common Lisp or purely backend-internal with no application-building guidance needed.
  - File: `skills/clim-spec/SKILL.md`

- **croatoan**
  - Build Common Lisp terminal applications with Croatoan using the right mix of screens, windows, event loops, bindings, forms, menus, dialogs, layouts, styling, and thread-safe terminal workflows.
  - Use when: the user wants to build, extend, debug, or broadly understand the practical feature set of Croatoan.
  - Do not use when: the question is generic terminal programming with no Croatoan angle, or purely raw ncurses work with no desire for Croatoan's higher-level API.
  - File: `skills/croatoan/SKILL.md`

- **coalton**
  - Build statically typed Common Lisp projects with Coalton using the right mix of ASDF/package setup, `coalton-toplevel`, algebraic data, type classes, Lisp interop, config, testing, debugging, and optimization.
  - Use when: the user wants to build, extend, debug, or broadly understand the practical feature set of Coalton.
  - Do not use when: the question is generic Common Lisp or generic FP with no Coalton angle.
  - File: `skills/coalton/SKILL.md`

- **common-lisp**
  - Build practical Common Lisp code using the Common Lisp Cookbook as a mirrored recipe corpus, with routing across its chapters and attached example files.
  - Use when: the user wants cookbook-grounded help for strings, loop, macros, CLOS, files/IO, sockets, FFI, threads, systems, Emacs IDE workflows, Win32, testing, or broad cookbook coverage.
  - Do not use when: the question is purely normative language-spec interpretation or has no Common Lisp cookbook angle.
  - File: `skills/common-lisp/SKILL.md`

- **guix**
  - Define, build, lint, and debug GNU Guix packages; create and maintain Guix channels; write system/service configurations; work with G-expressions, build phases, origins, and the full packaging workflow.
  - Use when: the user wants to package software for Guix, create a channel, customize build phases, write G-expressions, use `guix import`/`guix lint`/`guix build`, or broadly understand the Guix packaging model.
  - Do not use when: the question is generic Guile Scheme with no Guix angle, or is about Nix/NixOS without Guix relevance.
  - File: `skills/guix/SKILL.md`
